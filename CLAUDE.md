# Svodka

Персональный дайджест новостей: собирает посты из Bluesky, фильтрует по интересам и составляет ежедневную сводку на русском языке.

---

# Агент дайджеста

## Задача агента

Когда пользователь просит собрать сводку или дайджест, выполни следующие шаги по порядку:

### 1. Собери данные

Запусти коллектор:

```bash
uv run python collect.py
```

Если команда завершилась с ошибкой — сообщи об этом и не продолжай.

### 2. Прочитай собранные данные

Найди файл `raw/YYYY-MM-DD.md` (сегодняшняя дата) и прочитай его целиком.

### 3. Прочитай интересы пользователя

Прочитай файл `interests.md`.

### 4. Отфильтруй и составь дайджест

- Оставь только посты, которые соответствуют интересам из `interests.md`
- Безжалостно выброси всё нерелевантное
- Сгруппируй по темам
- Для каждого поста: краткое резюме на русском (1-3 предложения) + ссылка на оригинал
- В начале дайджеста — 2-3 предложения о том, что сегодня происходит в целом

### 5. Сохрани дайджест

Сохрани результат в `digests/YYYY-MM-DD.md`.

Сообщи пользователю путь к файлу.

## Важно

- Не добавляй посты, которые не соответствуют интересам, даже если их мало
- Если релевантных постов совсем нет — честно напиши об этом
- Дайджест пишется на русском языке
- `interests.md` и `sources.yaml` пользователь редактирует сам — не меняй их

# Агент разработки

## Стек

- **Python 3.14**, менеджер пакетов — `uv`
- **atproto** — клиент Bluesky (AT Protocol)
- **python-dotenv** — загрузка переменных окружения из `.env`
- **pyyaml** — чтение конфигурации `sources.yaml`

## Архитектура

Двухступенчатый пайплайн:

```
[Bluesky API] ──> collect.py ──> raw/YYYY-MM-DD.md ──> [Агент дайджеста] ──> digests/YYYY-MM-DD.md
```

## Ключевые файлы

| Файл | Назначение |
|------|-----------|
| `collect.py` | Сборщик данных из Bluesky → `raw/` |
| `main.py` | Точка входа (пока минимальная) |
| `sources.yaml` | Конфигурация источников (редактирует пользователь) |
| `interests.md` | Интересы для фильтрации (редактирует пользователь) |
| `.env` | Учётные данные (не коммитить) |
| `raw/` | Сырые собранные посты |
| `digests/` | Готовые дайджесты |

## Окружение

Зависимости устанавливаются через `uv`:

```bash
uv sync
```

Нужен файл `.env` с переменными (см. `.env.example`):
- `BLUESKY_HANDLE`, `BLUESKY_APP_PASSWORD`

## Запуск

```bash
uv run python collect.py   # сбор данных
uv run python main.py      # точка входа
```

## Что не трогать

- `sources.yaml` и `interests.md` — пользователь редактирует сам
- `raw/` и `digests/` — генерируются автоматически, не коммитить
- `.env` — содержит секреты, не коммитить
